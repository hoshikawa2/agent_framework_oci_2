=====================================================================
CAPÍTULO 1 – O QUE SERÁ CONSTRUÍDO
=====================================================================

Este tutorial não é conceitual.

Ele foi elaborado a partir do projeto real agent_framework_oci.

O objetivo será entender como o projeto template funciona e como criar
um novo agente seguindo exatamente a arquitetura implementada.

O projeto já possui dois agentes configurados:

- telecom_contas
- retail_orders

Eles estão registrados em:

config/agents.yaml

Trecho encontrado no projeto:

default_agent_id: telecom_contas

agents:
  telecom_contas
  retail_orders

Ou seja, o framework suporta múltiplos agentes utilizando o mesmo núcleo.

=====================================================================
CAPÍTULO 5 – CRIANDO UM NOVO AGENTE
=====================================================================

Vamos criar:

seguros_agent

Passo 1

Adicionar em agents.yaml

agent_id: seguros_agent

Passo 2

Criar diretório:

config/agents/seguros_agent

Passo 3

Criar:

prompt_policy.yaml
guardrails.yaml
judges.yaml

Passo 4

Adicionar metadata:

domain: seguros

Neste momento o framework já reconhece a existência do agente.

CAPÍTULO 7 – CRIANDO NOVAS INTENTS
=====================================================================

Criar:

CAPÍTULO 8 – MCP NO PROJETO REAL
=====================================================================

identificado em main.py.

Ele solicita:

e o MCP Tool Router resolve a integração.

CAPÍTULO 11 – BUSINESS CONTEXT
=====================================================================

O projeto utiliza:

IdentityResolver
BusinessContext

identificados no main.py.

Nunca utilizar:

policy_id
customer_id

diretamente no agente.

Sempre utilizar:

customer_key
contract_key

e deixar o MCPParameterMapper resolver.
